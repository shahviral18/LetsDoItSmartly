## 🔴 CRITICAL / HIGH

### 1. `WorkspaceUserController.php` — `get()` missing billing entity scope
`list()` correctly scopes to domain_owner's `billing_entity_id`, but `get($id)` does not. A domain_owner can fetch any workspace user by ID across all organizations.  
**Severity:** HIGH

### 2. `InvoiceController.php` — `get()` missing billing entity scope
Same pattern — `list()` is scoped, `get($id)` is not. Any domain_owner can read any invoice by ID.  
**Severity:** HIGH

### 3. `DomainController.php` — `get()` missing billing entity scope
`list()` scopes to domain_owner's billing entity, `get($id)` does not. Domain_owner can fetch any domain record.  
**Severity:** HIGH

### 4. `BccController.php` — `get()` missing billing entity scope
BCC request detail (`/surveillance/:id`) does not verify the requesting domain_owner owns the record. Any authenticated user can read any BCC request by ID.  
**Severity:** HIGH

### 5. `backend/public/index.php` + `src/pages/profile/ProfilePage.tsx` — Profile API schema mismatch
Backend returns a single `name` field; frontend `ProfileData` interface expects `first_name` + `last_name`. PATCH also sends `first_name`/`last_name` but the `portal_users` table only has a `name` column. Profile page loads blank and saves silently fail or corrupt data for all user types.  
**Severity:** HIGH

### 6. `src/router/roleRoutes.ts` — All nav paths prefixed with `/app/`
Route paths in `roleNavItems` use `/app/dashboard`, `/app/users`, etc. but `App.tsx` routes are `/dashboard`, `/users`, etc. Any sidebar component consuming this file will produce broken links.  
**Severity:** HIGH

## 🟡 MEDIUM

### 7. `src/pages/profile/ProfilePage.tsx:68` — Silent failure on profile load
`.catch(() => {})` swallows all errors. If `GET /api/profile` fails, loading clears but all form fields are empty with no error shown — indistinguishable from a real empty profile.

### 8. `src/pages/profile/ProfilePage.tsx:86-99` — `pwSaving` stuck on validation failure
`setPwSaving(true)` is called before the passwords-match check. On early return due to validation failure, the `finally` block never runs, leaving the button permanently in loading state.

### 9. `src/pages/admin/GoogleLoginHistoryPage.tsx` — `?email=` auto-open effect has empty dep array
The effect reading `searchParams.get('email')` runs only on mount (empty `[]` dep array). If the component is already mounted and the URL gets `?email=` (e.g., navigating from User Detail), the auto-open won't fire.

### 10. `src/context/DomainContext.tsx` — Re-fetch depends only on `user?.id`, not `user?.role`
If a user's role changes server-side, the domain list won't re-scope (`/my/domains` vs `/domains`) until a full page refresh. Effect dep array should include `user?.role`.

### 11. `src/pages/surveillance/BccRequestDetailPage.tsx` — Fragile null guard before property access
Code checks `if (!req) return` then immediately accesses `req.id`. Safe today but any async state update between guard and access would throw. Should destructure after the guard.

### 12. `src/pages/admin/SecurityAlertsPage.tsx` — `exportCSV` called with double-wrapped rows
`rows.map(r => [r])` wraps each already-array row in another array, producing a 3D structure instead of 2D. CSV export will produce malformed output.

### 13. `src/pages/domain/DomainDashboardPage.tsx` — Misleading empty state
When a domain_owner has no `billing_entity_id` assigned in DB, page shows "No domains found. Run Google Sync first." The actual problem is a missing BE assignment, not sync status.

### 14. `src/pages/admin/AuditLogPage.tsx` — Date range filter uses string comparison
`entry.created_at > dateTo + 'T23:59:59Z'` is lexicographic, not a real date comparison. Breaks if timestamps have timezone offsets or non-ISO formats. Should use `new Date()` objects.

## 🟢 LOW

### 15. `src/pages/profile/ProfilePage.tsx` — `domains` field missing for staff in backend response
Backend omits `domains` for staff users. Interface marks it optional so no crash, but the section silently never renders for staff. Undocumented behaviour.

### 16. `backend/controllers/DomainController.php:update()` — `name` not in allowed update fields
The update method only allows `ou_path` and `is_active`. Domain name cannot be corrected after creation without a direct DB edit.

### 17. `backend/services/GoogleWorkspaceService.php:listSharedDrives()` — Wrong Drive scope
`drives->listDrives()` and `permissions->listPermissions()` need `https://www.googleapis.com/auth/drive` (full), not `drive.readonly`. Will return 403 until the correct scope is added to domain-wide delegation in Google Admin console.

### 18. `src/pages/users/SharedDrivesPage.tsx` — `sizeMb` always 0
Backend `listSharedDrives()` hardcodes `sizeMb: 0` — the Drive API doesn't expose total shared drive size in the drives list endpoint. UI shows "0 GB" for every drive.
